How It Works
  1. Customer visits cart page Checkbox automatically appears: "Receive order updates in Beeylo app"
  2. Customer opts in Preference saved as cart attribute
  3. Order created Webhook sent to Beeylo with preference
  4. Notification delivered App notification + push (or email if not opted in)
Testing
  1. Visit your cart page You should see the "Receive in Beeylo app" checkbox
  2. Check the box and complete checkout Use an email linked to a Beeylo user
  3. Verify notification in Beeylo app Order should appear in app (no email sent)
Email Suppression (Shopify Plus Only)
Note: Shopify Flow is only available for Plus merchants. Use it to suppress emails when customers opt for app delivery.

Setup Workflow in Shopify Flow:

  1. Open Flow Admin → Apps → Flow → Create workflow
  2. Set Trigger Trigger: Order created
  3. Add Condition IF Note attributes contains "Receive_in_Beeylo_App" AND "Yes"
  4. Add Action Add tag: "beeylo-app-delivery"
Order Created ↓ Check: note_attributes = "Receive_in_Beeylo_App: Yes" ↓ [TRUE] → Tag order: "beeylo-app-delivery"
Non-Plus Alternatives:
• Disable all emails in Settings → Notifications
• Use third-party app (Klaviyo, Omnisend)
• Accept duplicate notifications
Troubleshooting
  • Checkbox not appearing? Check browser console for errors. Script should load from /public/beeylo-cart-integration.js
  • Notifications not delivering? Verify customer has beeylo_user_id linked in database
  • Need more help? Check Railway logs or review webhook events in dashboard
